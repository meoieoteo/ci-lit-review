# Specialist Review: EHR Schema and Common Data Model

## Specialist Scope
Assessment of EHR text sources, validation, and portability in an ageing-syndrome NLP systematic review.

## Bottom Line
Moderate schema relevance: the review catalogs diverse EHR text sources and validation weaknesses, but it does not provide a unified Epic or CDM pipeline.

## Data Model and Source System Summary
Studies used heterogeneous EHR text sources from multiple clinical settings.

## Unit of Analysis and Identifiers
Likely varies by study: note, encounter, admission, or patient.

## Encounter and Timeline Representation
Variable and often not central; source checks are required for timing claims.

## Cohort and Eligibility Implementation
Study-specific eligibility across ageing syndromes.

## Structured Feature Representation
Some studies include structured EHR context, but clinical text is the core focus.

## Note and Text Metadata Representation
The review is useful for enumerating note types but not for standardized note metadata implementation.

## Label and Outcome Data Representation
Phenotype labels and reference standards vary across studies.

## Vocabulary, Code Sets, and Mappings
No unified CDM vocabulary mapping.

## Missingness, Data Quality, and Provenance
Validation quality and reference-standard limitations are important data-quality concerns.

## Reproducibility and Portability
External validation and transportability are limited across the field.

## Fit to Epic-to-Model-Ready Pipeline
Moderate as a cautionary source for note-type and validation planning, low as an implementation template.

## Strengths
- Broad map of EHR note sources.
- Quality assessment.

## Concerns
- Heterogeneous methods and reporting.
- No single reproducible pipeline.

## Missing Information
- Exact schema details for high-value included studies.

## Implications for This Literature Review
Use to justify explicit note-source, reference-standard, and validation reporting in the local pipeline.

## Recommended Follow-up
Select individual included studies for source-level schema extraction.
